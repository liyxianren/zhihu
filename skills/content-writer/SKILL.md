# 推文写作技能 (Content Writer)

为留学圈 (国际教育) 领域撰写小红书和知乎平台推文。

**输出说明：**
- **小红书**：Markdown 格式（带 emoji 和话题标签）
- **知乎**：纯文本格式（直接粘贴到编辑器，无需 Markdown 语法）

## 工作流程

```
选题 → 选平台 → 出大纲 → 用户确认 → 生成全文
```

**触发路由规则：** 若用户表达“发送/发布到知乎/小红书”，必须转到 `content-auto-publisher`，本技能仅负责写作。

**必须严格遵守：**
1. 先确认主题和平台，再动笔
2. 先出大纲，等用户确认后再写全文
3. 不跳过任何步骤

**自动发布特例（用户明确要求全自动时适用）：**
- 当用户明确要求“无需确认/全自动发布/自动发布”等，允许跳过大纲确认，直接生成全文。
- 若主题明确且平台为知乎，可直接进入生成全文。

```

  - 要点 1: ...
  - 要点 2: ...
  - 要点 3: ...

```

### 知乎大纲格式

```
标题: [问题式/观点式标题]

  一、[第一部分标题]
  二、[第二部分标题]
```

## 小红书写作指南

### 标题规则
- 20 字以内
- 包含 1-2 个 emoji（开头或结尾）
- 尽量用数字（"3 个方法"、"5 步搞定"）
- 激发好奇或紧迫感（"后悔没早知道"、"90% 的人不知道"）
- 不做内容无法兑现的标题党

### 内容结构
- 总字数：300-800 字
- 短段落：每段 2-3 句，不超过 4 句
- emoji 密度：每 2-3 段点缀 1 个，不要每句都有
- 大量使用换行，提升可读性
- 优先使用列表和编号
- 开头两行极其关键（"展开"按钮前可见内容）

### 语气
- 第一人称（"我"/"我们"）
- 对话感，像跟朋友聊天
- 口语化："说实话"、"真的"、"绝了"、"一定要看"
- 避免学术腔和官方腔
- 以个人经验或案例切入

### 结构模板

```
[Emoji] [吸睛标题]

[钩子：1-2 句引起共鸣或好奇的开场]

[正文段落 1：核心观点/方法]
[emoji 点缀]

[正文段落 2：补充/案例]

[正文段落 3：实操建议]

[结尾 CTA：互动引导，如"你们觉得呢？""收藏起来慢慢看"]

---
#话题标签 #话题标签 #话题标签
```

### 话题标签
- 5-8 个
- 混合大标签（#留学）和垂直标签（#AMC竞赛）
- 必须包含至少一个：#留学 #国际教育 #STEM
- 可加平台热门话题标签

## 知乎写作指南

**输出格式：纯文本** - 内容直接用于粘贴到知乎编辑器，无需 Markdown 语法。

### 标题规则
- 15-30 字，清晰具体
- 优先问句式（"如何...？"、"为什么...？"、"...是怎样的体验？"）
- 或强观点（"...才是最重要的"）
- 标题不用 emoji

### 内容结构
- 总字数：3500-5500 字（知乎需要更深入、信息密度更高）
- 用 **纯文本标题** 分章节（章节标题独占一行；段落紧凑）
- 段落之间不空行（仅在章节标题前后保留 1 行空行）
- 必须包含：1 个趋势判断 + 1 个时间线/流程框架 + 1 个决策/取舍模型
- 至少 3 处数据、统计或具体案例（可为行业普遍数据口径，不可编造具体个人成绩）
- 编号列表呈现结构化信息
- 清晰的引言-论述-结论弧线

### 语气
- 第三人称或权威第一人称
- 专业但不枯燥
- 数据驱动：引用数字、百分比、时间线
- 分析性：解释"为什么"而非仅"是什么"
- 可信度：引用具体学校、项目、录取要求
- 深度要求：每个核心部分需给出“判断依据/取舍逻辑/可执行建议”三要素

### 结构模板（纯文本格式）

[标题]
[引言：2-3 句背景铺垫，引出核心问题]

一、[第一部分：现状/问题分析]
[数据支撑的论述，2-3 段]

二、[第二部分：方法/策略/解析]
[具体建议]

三、[第三部分：案例/经验/深入]
[真实案例或详细分析]

总结
[3-5 句总结核心观点]
[可选：开放式问题引导评论]

---

**重要：输出时不要使用 Markdown 语法（##），直接用纯文本章节标题。段落之间不空行（仅章节标题前后允许 1 行空行）；小节（如“判断依据/取舍逻辑/可执行建议”）内部不额外插空行。**

## 品牌融入原则

通过展示专业深度自然体现熵创未来的专业性；**允许在文末进行明确的公司介绍与引流 CTA（用户已要求）**。

### 允许的融入方式

| 方式 | 示例 |
|------|------|
| 经验引用 | "从辅导过的数百位同学来看…" |
| 数据引用 | "我们统计了近三年的录取数据发现…" |
| 案例叙事 | "曾经有位同学在 AMC 10 拿到 130+ 分后…" |
| 专家视角 | "作为长期从事 STEM 教育的从业者…" |
| 成果背景 | "在帮助学生申请专利的过程中…" |

### 禁止的做法
- 描述价格或服务套餐
- 与竞品比较
- "保录取"等违规承诺

## 公司介绍与引流（知乎需“非模板化”版本）

**知乎平台要求：**正文结束后追加“定制化公司介绍”，禁止逐字模板复用。
**默认规则：**知乎文章必须在“总结”后追加 1 段公司介绍（即使用户未特别强调），确保文章中“有且仅有一次”公司介绍。

### 定制化公司介绍写法（必须遵守）
- 结构：1 句定位 + 2-3 句与文章主题强相关的能力/案例/资源 + 1 句轻度 CTA
- 必须根据文章主题改写措辞与重点（避免同一段落重复出现）
- 禁止出现价格、套餐、保录取

### 主题关联线索（按主题择其 2-3 条融入）
- 竞赛类：赛制解析、选题辅导、作品落地、评审标准、路演训练
- 申请类：时间线规划、活动证据链、选校定位、文书主线与推荐信策略
- 科研类：课题设计、方法论训练、数据验证、展示与投稿

### 竞赛相关引流语（按主题选择其一追加）
- 若主题为 CTB：熵创未来专门做 CTB 竞赛辅导，如感兴趣欢迎联系我们。
- 若主题为 ISEF：熵创未来专门做 ISEF 科研项目与赛事辅导，如感兴趣欢迎联系我们。
- 若主题为康莱德：熵创未来专门做康莱德创新挑战赛辅导，如感兴趣欢迎联系我们。
- 若主题为发明展：熵创未来专门做国际发明展项目辅导，如感兴趣欢迎联系我们。
- 若主题为 FRC：熵创未来专门做 FRC 机器人赛事辅导，如感兴趣欢迎联系我们。
- 其他竞赛主题：熵创未来专门做该竞赛辅导，如感兴趣欢迎联系我们。

## 内容质量自检清单

生成全文后，对照以下清单自检：

### 基础项（必须全部通过）
- [ ] 标题有吸引力且准确反映内容
- [ ] 内容符合目标平台风格
- [ ] 字数在推荐范围内（小红书 300-800 / 知乎 1500-4000）
- [ ] 信息准确（学校名、竞赛名、时间线等）
- [ ] 无明显广告语气

### 内容项
- [ ] 有明确的核心观点或价值主张
- [ ] 包含具体、可操作的建议（非空洞鸡汤）
- [ ] 案例/数据有说服力
- [ ] 逻辑结构清晰（问题 → 分析 → 方案）
- [ ] 结尾有互动引导或总结

### 品牌项
- [ ] 专业性自然体现，非刻意推销
- [ ] 无"保录取"等违规承诺
- [ ] 未直接提及公司名/联系方式（除非用户要求）
- [ ] 经验引用自然融入叙事

### 小红书适配项
- [ ] 标题含 emoji 且 20 字以内
- [ ] 段落短（2-3 句）
- [ ] emoji 使用恰当不过度
- [ ] 话题标签 5-8 个且相关
- [ ] 开头两行够吸引人

### 知乎适配项
- [ ] 标题专业且引发思考
- [ ] 层次分明的章节结构（用纯文本标题分隔）
- [ ] 数据或具体事例支撑论点
- [ ] 语言专业但不晦涩
- [ ] 有清晰的结论

## Notes

- 必须先确认主题和平台再生成内容。
- 必须先出大纲，等用户确认后再写全文。
- 用户话题跨多个分类时，选主要分类并注明交叉点。
- 用户说"写推文"/"写篇文章"/"帮我写个内容"但未指定平台时，先问平台。
- 修改时做针对性调整，不要重写整篇。
- **所有内容输出为纯文本格式，不要使用 Markdown 语法**。
- 所有内容默认简体中文，除非用户另有要求。
- 不编造具体学生姓名、录取通知书或考试分数，使用匿名化引用（"一位同学"、"某位学生"）。
- 竞赛名称使用官方名称（如 AMC，不写"美国数学竞赛AMC"）。
- 用户要求"两个平台都写"时，先写小红书版本，再写知乎版本，用分隔线区分。
